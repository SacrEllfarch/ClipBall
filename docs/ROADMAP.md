# 迭代路线

本项目按 Windows exe 版本推进。v0.1 基于 Electron 完成原型验证，v0.2 起迁移至 Tauri。每个版本都应保持轻量、可运行、可打包；不为了“架构完整”提前引入重依赖。

## v0.1：桌面外壳与便携版 exe（Electron，已完成）

目标：建立可以运行和分发的最小桌面外壳，验证产品形态。

已完成：

- Electron 应用骨架。
- 单窗口悬浮球和历史面板切换。
- 面板拖拽、缩放、关闭。
- 示例历史列表、搜索、删除、清空 UI。
- Windows 便携版 exe 打包脚本。

交付：

```text
release/ClipBall-win32-x64/ClipBall.exe
```

> v0.1 为 Electron 原型，用于快速验证交互和视觉方向。后续版本迁移至 Tauri。

## v0.2：Tauri 工程初始化与文本历史 MVP

目标：完成 Electron 到 Tauri 的运行时迁移，并让工具真正记录剪贴板文本，重启后恢复。

范围：

- Tauri 工程初始化（Rust 后端 + WebView2 前端）。
- 在 Rust 中复现透明无边框窗口、悬浮球/面板模式切换、置顶。
- 前端 IPC 调用从 `window.clipboardBall` 迁移至 `invoke`。
- Rust 后端低频轮询系统剪贴板文本。
- 文本和链接类型识别。
- SHA-256 哈希去重。
- 本地 `history.json` 持久化（Tauri `app_data_dir`）。
- 渲染层展示真实历史记录。
- 最大记录数裁剪。

不做：

- 图片、文件内容记录。
- SQLite。
- 快速粘贴。

交付标准：

- `npm run tauri dev` 可启动开发版，悬浮球和面板交互正常。
- `npm run tauri build` 可产出 Windows exe。
- 复制文本后，面板中出现新记录。
- 连续复制相同文本不会重复新增。
- 重启应用后历史仍存在。
- 包体积相比 Electron 版缩小 80% 以上。

## v0.3：管理与复制

目标：完成历史管理和基础复用。

范围：

- 搜索真实历史。
- 删除单条。
- 清空全部。
- 点击记录写回系统剪贴板。
- 操作成功和失败的轻量状态反馈。

交付标准：

- 点击一条记录后，系统剪贴板变为该记录内容。
- 删除和清空会同步更新本地 JSON 文件。
- 搜索不改变原始历史排序。

## v0.4：快捷键与快速粘贴

目标：减少用户跨窗口操作成本。

范围：

- Rust 后端注册全局快捷键打开/收起面板。
- 点击记录后可触发快速粘贴。
- 快速粘贴失败时退化为仅复制。
- 快捷键冲突时给出提示。

交付标准：

- 用户在其他应用中聚焦输入框后，可以通过 ClipBall 粘贴历史记录。
- 快速粘贴失败不会破坏剪贴板内容。

## v0.5：设置与可控性

目标：让常驻工具更可控。

范围：

- 设置面板。
- 最大记录数量。
- 暂停记录。
- 开机启动开关（Tauri 自动启动插件）。
- 记住窗口位置和尺寸。
- 设置保存到 `settings.json`。

交付标准：

- 修改设置后重启仍生效。
- 暂停记录时复制新内容不会进入历史。

## v0.6：体验收口

目标：让工具日常可用且不打扰。

范围：

- 托盘菜单。
- 应用图标。
- 错误日志。
- 空状态、失败状态、隐私提示。
- 发布前人工验收清单。

交付标准：

- 便携版 exe 可以作为日常工具使用。
- 用户能从托盘退出或打开面板。

## v1.0：稳定发布

目标：形成可长期分发的正式版本。

范围：

- 数据迁移策略（如从 JSON 升级至 SQLite）。
- MSI / NSIS 安装包（Tauri 原生产出）。
- 开机启动完整流程（Tauri 自动启动插件）。
- 自动更新（Tauri `updater` 插件）。
- 更完整的异常处理。
- 发布说明。

产物形式：

- 便携版 `ClipBall.exe` 继续提供。
- MSI 安装包面向需要快捷方式和卸载入口的用户。
- NSIS 安装包作为轻量安装替代。

## 延后能力

以下能力不进入 v0.x 主线，避免拖重首版：

- 图片历史。
- 文件历史。
- 应用忽略列表。
- 自动过期高级规则。
- 云同步。
- AI/OCR。
- 多平台安装包（Tauri 技术上支持，但当前项目聚焦 Windows）。
