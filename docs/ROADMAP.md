# 迭代路线

## Phase 0：文档与原型整理

目标：明确产品范围、技术方案和实现边界。

- 已完成：整理 AGENTS、README、PRD、SDD。
- 保留现有 `剪贴板历史记录工具.html` 作为视觉基准。
- 确认首版桌面运行时，默认建议 Electron。

## Phase 1：工程初始化

目标：建立可运行的桌面应用骨架。

- 初始化包管理与构建脚本。
- 建立 Electron 主进程、preload、renderer 目录。
- 迁移现有 HTML 原型到 renderer。
- 应用启动后显示悬浮球。
- 点击悬浮球展开面板，关闭后回到悬浮球。

交付标准：本地运行后可以看到与原型接近的悬浮球和历史面板。

## Phase 2：文本历史 MVP

目标：完成文本剪贴板历史闭环。

- 主进程轮询剪贴板文本。
- 对内容做哈希去重。
- 写入本地 SQLite。
- UI 展示真实历史记录。
- 支持删除单条和清空全部。
- 重启后恢复历史。

交付标准：复制文本后出现在历史面板中，并能被删除和持久保存。

## Phase 3：复制与快速粘贴

目标：让历史记录真正服务当前工作流。

- 点击记录写回系统剪贴板。
- 支持点击记录后自动粘贴。
- 注册打开面板快捷键。
- 预留粘贴最近一条快捷键。
- 快速粘贴失败时退化为仅复制。

交付标准：用户在其他应用中聚焦输入框后，可以通过面板或快捷键粘贴历史记录。

## Phase 4：设置与体验完善

目标：让用户可控、可配置。

- 设置最大记录数量。
- 设置快捷键。
- 暂停记录。
- 收藏记录。
- 搜索过滤。
- 记住窗口位置和尺寸。
- 支持开机启动开关。

交付标准：用户可以按自己的习惯配置工具，常用设置重启后仍生效。

## Phase 5：多类型内容

目标：扩展到文本之外。

- 链接类型识别和展示。
- 图片缩略图和体积限制。
- 文件路径记录和展示。
- 按类型筛选。

交付标准：面板可以清晰区分文本、链接、图片、文件记录。

## Phase 6：发布准备

目标：达到可分发状态。

- 应用图标和托盘菜单。
- 安装包配置。
- 数据库迁移机制。
- 错误日志。
- 发布前人工验收清单。
- Windows 优先打包，其他平台后续补齐。

交付标准：可生成安装包，并通过核心功能验收。
