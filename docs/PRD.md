# 产品需求说明 PRD

## 1. 背景

用户在写作、开发、整理资料或跨应用操作时，经常需要反复使用近期复制过的内容。系统自带剪贴板历史通常入口固定、定制能力有限，且无法很好地贴合用户的悬浮工作流。

本工具希望提供一个轻量常驻的剪贴板历史入口：不使用时是悬浮球，需要时展开为历史面板，用户可以通过鼠标或快捷键快速粘贴历史记录。

## 2. 用户目标

- 快速找回近期复制过的文本、链接、文件或图片。
- 减少频繁切换窗口和重复复制的成本。
- 在当前输入位置快速粘贴指定历史内容。
- 本地保存历史，避免隐私数据离开设备。
- 可以删除不需要或敏感的历史记录。

## 3. 目标用户

- 高频复制粘贴的办公用户。
- 开发、运营、设计、客服等需要整理大量片段信息的用户。
- 需要跨应用搬运内容且希望保持轻量入口的桌面用户。

## 4. 核心场景

### 4.1 打开历史面板

用户看到屏幕右下角悬浮球，点击后展开为历史面板。面板默认显示最新记录，顶部提供搜索，底部显示记录数量和清空入口。

### 4.2 快速粘贴记录

用户在任意应用中聚焦输入位置，打开历史面板，点击一条历史记录。工具将该记录写回系统剪贴板，并触发粘贴动作。

### 4.3 使用快捷键粘贴

用户按下设置好的全局快捷键打开历史面板或粘贴最近一条记录。快捷键应可配置，并避免与系统常用快捷键冲突。

### 4.4 管理历史

用户可以删除某一条记录，也可以清空全部记录。收藏记录不应被普通自动清理策略删除，除非用户明确清空收藏。

## 5. 功能需求

### 5.1 悬浮球

- 默认显示在屏幕右下角。
- 可点击展开历史面板。
- 后续可支持拖动悬浮球位置并记住位置。
- 应保持置顶，但不遮挡太多内容。

### 5.2 历史面板

- 点击悬浮球展开。
- 支持拖拽移动。
- 支持调整大小。
- 支持关闭后回到悬浮球。
- 支持固定面板状态。
- 列表按时间倒序展示。

### 5.3 剪贴板记录

- 监听剪贴板变化。
- 识别文本、链接、图片、文件等类型。
- 首版优先支持文本和链接。
- 对连续重复内容去重。
- 记录时间、类型、预览内容和原始内容。
- 支持最大记录数量限制。

### 5.4 本地存储

- 所有历史默认保存到本地。
- 重启应用后历史仍可恢复。
- 支持删除单条和清空全部。
- 支持后续自动过期策略。

### 5.5 搜索

- 用户输入关键词后即时过滤历史记录。
- 首版按文本内容和链接 URL 搜索。
- 搜索不改变原始排序。

### 5.6 快速粘贴

- 点击记录时，工具应写回剪贴板。
- 当启用快速粘贴时，工具应向当前活动应用发送粘贴命令。
- 如果系统不允许模拟粘贴，则至少完成写回剪贴板，并给出 UI 状态反馈。

### 5.7 设置

首版预留设置入口，后续支持：

- 快捷键配置。
- 最大记录数量。
- 是否开机启动。
- 是否记录图片和文件。
- 自动清理周期。
- 暂停记录。
- 忽略应用列表。

## 6. 非功能需求

- 启动速度快，常驻资源占用低。
- 默认本地运行，不联网。
- 剪贴板轮询应节流，避免高 CPU 占用。
- 面板交互应流畅，无明显卡顿。
- 敏感内容需要用户可控删除。
- 数据结构可迁移，便于后续版本升级。

## 7. MVP 范围

MVP 只覆盖最小可用闭环：

- 悬浮球。
- 历史面板。
- 文本和链接记录。
- 本地持久化。
- 点击记录写回剪贴板。
- 快捷键打开面板。
- 删除单条和清空全部。

不进入 MVP：

- 云同步。
- OCR。
- 多设备同步。
- 复杂规则引擎。
- 图片和文件的完整处理。
- 账号系统。

## 8. 验收标准

- 复制一段文本后，面板中出现新记录。
- 连续复制相同文本不会产生重复记录。
- 重启应用后历史仍存在。
- 点击记录后，剪贴板内容变为该记录。
- 开启快速粘贴后，点击记录可以在当前输入位置粘贴。
- 删除单条后，该记录从列表和本地存储中消失。
- 清空后，历史列表为空。
- 面板可以拖拽和缩放。
