# 剪贴板历史记录工具

一个桌面端剪贴板历史小工具。默认以悬浮球常驻屏幕，点击后展开为可拖拽、可调整大小的历史面板。用户可以查看本地复制历史，点击记录或使用快捷键完成快速粘贴。

当前仓库已有一个 UI 原型：[剪贴板历史记录工具.html](Z:/project/ball/剪贴板历史记录工具.html)。后续实现应以该原型为视觉参考。

## 目标体验

- 未展开时：显示一个轻量、可见、不打扰的悬浮球。
- 展开后：显示历史面板，支持拖拽位置和调整大小。
- 复制后：自动记录文本、链接、文件、图片等剪贴板内容。
- 使用时：点击历史记录即可复制或快速粘贴到当前输入位置。
- 管理时：支持搜索、收藏、删除单条、清空全部、设置最大记录数。

## 建议技术栈

首版已采用 Electron：

- 复用现有 HTML/CSS/JS 原型成本低。
- 支持系统剪贴板读取与写入。
- 支持托盘、全局快捷键、窗口置顶、无边框悬浮窗口。
- 方便接入 SQLite 或本地文件存储。

后续如果更关注包体积，可以评估 Tauri。

## 本地运行

安装依赖：

```powershell
npm install
```

启动桌面应用：

```powershell
npm start
```

运行基础语法检查：

```powershell
npm run check
```

## MVP 范围

第一阶段建议先完成文本历史的闭环：

1. 启动应用并显示悬浮球。
2. 点击悬浮球展开历史面板。
3. 监听文本剪贴板变化并写入本地数据库。
4. 在面板中展示历史记录。
5. 点击记录写回剪贴板并触发粘贴。
6. 支持删除单条和清空全部。

图片、文件、复杂隐私规则和同步能力不进入首个 MVP。

## 隐私原则

本工具默认只在本地保存数据，不上传、不同步、不调用外部服务。用户应能够随时暂停记录、删除单条记录、清空全部历史，并配置最大保存数量或自动过期策略。
