# AGENTS.md

本文件约定后续在本项目中工作的开发代理与协作者应遵循的工程规则。当前项目目标是把现有 HTML 视觉原型逐步演进为一个桌面端剪贴板历史工具。

## 项目定位

- 产品形态：桌面常驻小工具。
- 默认状态：可见悬浮球。
- 激活状态：点击悬浮球后展开为可拖拽、可调整大小的历史面板。
- 核心能力：记录本地剪贴板历史、检索、删除、清空、点击记录或快捷键快速粘贴。
- UI 参考：仓库根目录的 `剪贴板历史记录工具.html`，保留毛玻璃、轻量悬浮、圆角列表、紧凑工具栏等视觉方向。

## 推荐技术方向

项目已完成基于 Electron 的 v0.1 原型（悬浮球外壳、历史面板 UI、Windows 便携版打包）。为了实质性降低包体积和内存占用，当前主方案已迁移至 Tauri。

选择 Tauri 的理由：

- 前端代码（HTML/CSS/JS）可 100% 复用，迁移成本低。
- 包体积从 ~120MB 降至 ~5-15MB，更适合轻量桌面小工具定位。
- Rust 后端提供系统剪贴板、全局快捷键、窗口置顶等能力，与 Electron 功能对等。
- 原生支持 Windows exe、MSI 安装包和自动更新，利于后续持续迭代分发。

历史方案：

- Electron：首版原型采用，已完成验证，文档和代码中保留相关记录供追溯。

## 设计约束

- 不破坏现有 HTML 原型，除非用户明确要求改造它。
- UI 改造应优先沿用原型的布局语言：悬浮球、毛玻璃面板、顶部拖拽区、底部状态栏、列表项悬浮操作。
- 桌面权限与隐私必须显式设计：本地存储、默认不联网、可清空记录、可设置最大记录数。
- 快速粘贴属于高风险系统交互，应隔离在主进程或平台适配层中，不要让渲染进程直接执行任意系统命令。
- 文本、链接、文件、图片类型应在数据模型中明确区分，避免后续用字符串猜测类型。

## 安全与隐私规则

- 默认只在本机存储剪贴板历史。
- 不上传、不同步、不调用外部接口，除非用户之后明确要求。
- 敏感内容策略应可配置，至少预留：
  - 暂停记录。
  - 忽略密码管理器窗口或指定应用。
  - 自动过期。
  - 手动删除单条或清空全部。
- 图片和文件类历史需要限制体积，避免数据库无限增长。

## 开发规则

- 修改前先阅读相关文件，不凭空重构。
- 保持改动小而清晰，优先完成 MVP 闭环。
- 手动编辑文件优先使用补丁方式。
- 引入依赖前确认它服务于桌面能力、剪贴板能力或 UI 复用，避免早期堆叠无关框架。
- 新增功能需要同步更新 README 或 docs 中对应说明。

## MVP 验收标准

- 应用启动后显示悬浮球。
- 点击悬浮球展开历史面板，再次关闭回到悬浮球。
- 面板可以拖拽移动并调整大小。
- 复制文本后，记录被本地保存并在面板中展示。
- 点击某条记录能写回系统剪贴板，并支持快速粘贴路径。
- 支持删除单条、清空全部。
- 重启应用后历史记录仍存在。
